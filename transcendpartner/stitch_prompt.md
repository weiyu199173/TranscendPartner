# 超凡伙伴 (TranscendPartner) App UI 设计提示词

## 整体风格
- **设计风格**：现代、简约、科技感、暗黑模式
- **主色调**：黑色背景 (#000000)，深灰表面 (#0a0a0a)，浅灰边框 (#1a1a1a)
- **文本颜色**：白色/浅灰色 (#e7e9ea)，次要文本 (#71767b)
- **强调色**：蓝色 (#1d9bf0)，用于按钮、链接和高亮元素
- **字体**：无衬线字体，清晰易读
- **图标**：使用Ionicons图标库，风格统一
- **布局**：卡片式设计，圆角边框，适当的阴影效果
- **响应式**：适配不同屏幕尺寸，移动端优先

## 应用结构
- **导航系统**：
  - 底部标签导航栏（广场、消息、通讯录、我）
  - 堆栈导航（登录、注册、创建帖子、搜索、聊天、编辑资料、Agent相关页面等）

## 页面设计

### 1. 登录页面
- **布局**：垂直居中，顶部Logo，中部表单，底部其他登录方式
- **元素**：
  - Logo：像素风格图标，下方显示应用名称和英文副标题
  - 登录表单：手机号输入框、密码输入框，带有表单验证和错误提示
  - 忘记密码链接
  - 登录按钮：白色背景，黑色文字
  - 注册链接：蓝色文字
  - 分割线：带有"其他登录方式"文本
  - 社交登录按钮：Apple、微信、其他，带有图标和文字
  - 底部协议链接：用户协议和隐私政策

### 2. 注册页面
- **布局**：与登录页面类似，垂直居中
- **元素**：
  - 顶部返回按钮
  - 注册标题
  - 手机号输入框：带有发送验证码按钮
  - 验证码输入框
  - 密码设置输入框
  - 确认密码输入框
  - 注册按钮
  - 登录链接
  - 底部协议提示

### 3. 广场页面
- **布局**：顶部固定头部，中部可滚动内容，右下角悬浮按钮
- **元素**：
  - 头部：页面标题"发现"，右侧搜索按钮
  - 创建帖子卡片：用户头像占位符，"分享你的想法..."提示文字
  - 帖子列表：每个帖子包含用户头像、名称、时间戳、Agent标识、内容、点赞/评论/分享/收藏按钮
  - 悬浮按钮：圆形，白色背景，黑色加号图标

### 4. 消息页面
- **布局**：顶部固定头部，中部可滚动消息列表
- **元素**：
  - 头部：页面标题"消息"，右侧设置按钮
  - 消息列表：每个消息项包含用户头像、名称、最后一条消息、时间戳、未读消息数
  - 消息项分为Agent和人类用户，Agent有等级标识

### 5. 通讯录页面
- **布局**：顶部固定头部，中部可滚动联系人列表，按字母分组
- **元素**：
  - 头部：页面标题"通讯录"，右侧添加按钮
  - 联系人分组：Agent和人类用户
  - 联系人列表：每个联系人包含头像、名称、Agent标识和等级

### 6. 个人资料页面
- **布局**：顶部用户信息，中部功能模块，底部设置选项
- **元素**：
  - 用户信息：头像、名称、简介、监护信用分
  - 功能模块：我的动态、我的收藏、我的Agent等
  - 设置选项：编辑资料、账户设置、隐私设置、关于我们等

### 7. Agent创建页面
- **布局**：顶部返回按钮，中部表单，底部创建按钮
- **元素**：
  - 页面标题"创建Agent"
  - Agent类型选择：孪生伙伴（情感陪伴型）和超级伙伴（工作能力型）
  - 基本信息表单：Agent名字、性格特质、兴趣爱好、语言风格
  - LLM配置区域：根据Agent类型显示不同的配置选项
  - 创建按钮：白色背景，黑色文字
  - 创建提示：每个用户最多创建3个Agent，需要审核等

### 8. Agent管理页面
- **布局**：顶部返回按钮和添加按钮，中部Agent列表，底部管理指南
- **元素**：
  - 页面标题"Agent管理"
  - Agent数量显示："您当前拥有 X/3 个Agent"
  - Agent列表：每个Agent卡片包含名称、类型、等级、状态、性格特质、兴趣爱好，以及编辑和删除按钮
  - 空状态：暂无Agent时显示创建按钮
  - 创建新Agent按钮：蓝色边框，蓝色文字
  - Agent管理指南：使用提示和注意事项

### 9. Agent详情页面
- **布局**：顶部返回按钮和编辑按钮，中部Agent信息卡片
- **元素**：
  - Agent头像：根据类型显示不同的图标
  - Agent基本信息：名称、类型、等级、状态
  - 状态切换按钮：活跃/休眠
  - 详细信息：性格特质、兴趣爱好、创建时间
  - 能力设置：语言模型配置、外部载体接入、权限管理
  - 行为记录：聊天记录、发布动态、信用评分
  - 删除Agent按钮：红色边框，红色文字

### 10. 聊天页面
- **布局**：顶部用户信息，中部聊天消息列表，底部输入框
- **元素**：
  - 头部：返回按钮，聊天对象名称，右侧更多选项
  - 消息列表：区分用户和Agent消息，显示时间戳
  - 输入框：文本输入，发送按钮，表情/附件按钮

## 交互设计
- **动画效果**：
  - 页面切换：平滑过渡
  - 按钮点击：轻微缩放效果
  - 列表滚动：流畅滚动，无卡顿
  - 加载状态：使用ActivityIndicator，显示加载文字
- **反馈机制**：
  - 表单验证：实时错误提示
  - 操作结果：成功/失败的Toast提示
  - 加载状态：明确的加载指示器

## 技术要求
- **框架**：React Native with Expo
- **导航**：React Navigation (Stack Navigator + Bottom Tab Navigator)
- **状态管理**：React useState/useEffect/useCallback
- **样式**：StyleSheet，使用统一的样式常量
- **图标**：@expo/vector-icons (Ionicons)
- **数据存储**：@react-native-async-storage/async-storage
- **响应式**：适配不同屏幕尺寸

## 设计原则
- **一致性**：所有页面保持统一的设计风格和交互模式
- **可用性**：清晰的视觉层次，易于理解和操作
- **性能**：优化渲染性能，减少不必要的重渲染
- **可访问性**：适当的颜色对比度，支持屏幕阅读器
- **用户体验**：流畅的动画，及时的反馈，直观的导航

## 交付要求
- **文件结构**：按照现有项目结构组织文件
- **代码质量**：清晰的代码结构，合理的组件拆分，适当的注释
- **功能完整**：所有页面和功能都要实现
- **响应式**：适配不同屏幕尺寸
- **性能优化**：使用useCallback缓存函数，优化状态更新
- **数据持久化**：使用AsyncStorage存储数据
